# INF-TIMESTAMPER 実装計画

`docs/要件.md` を権威ある正本として、C# / .NET 8 / WPF による実装をフェーズに分割した計画書。

**役割**: Claude Code セッションが切れても次セッションが文脈を引き継げるようにする、進捗と「次にやること」の永続化。各フェーズの完了基準（DoD）を明文化する。

進捗を更新する際は、各フェーズの DoD のチェックを埋め、必要なら新しい派生タスクを追記する。

## 全体方針

- 要件.md と本ファイルが食い違った場合、**要件.md を優先**する（本ファイルは作業計画であり、仕様の正本ではない）。
- フェーズ境界では「ビルドが通り、テストが通り、最低限の実機確認ができる」状態を維持する。
- 大きな抽象化は使用箇所が出てから導入する（YAGNI）。例：「ゲーム抽象化」も MVP では INFINITAS 専用で書き、2 つ目のゲーム追加時に抽出する。

## プロジェクト構成

```
inf-timestamper/
├── InfTimestamper.sln
├── src/
│   ├── InfTimestamper/                  WPF アプリ本体（UI / ViewModel / DI 組立）
│   └── InfTimestamper.Core/             ドメイン層（UI 非依存。データモデル、状態機械、永続化、画像認識、OBS 抽象）
└── tests/
    └── InfTimestamper.Core.Tests/       xUnit テスト
```

- WPF プロジェクトは Core を参照する。Core は WPF を参照しない。
- 画像認識・OBS 接続・JSON 永続化は Core に置き、UI からはサービス境界経由で呼ぶ（テスタビリティと 2 台 PC 構成オプションの両立）。
- ViewModel は WPF プロジェクトに置く（`System.ComponentModel.INotifyPropertyChanged` の参照範囲を WPF 側に閉じる方が後々楽）。状態射影ロジックなど純粋計算は Core にも分離可能。

### Phase 2 — コアドメイン

**ゴール**: 状態機械とデータモデルがコードで動き、JSON で読み書きできる。GUI なしでも振る舞いがテストできる。

**作業**:
- [x] データモデル定義（`Models/GameId` / `Models/AppInfo` / `Models/StreamInfo` / `Models/TimestampEntry` / `Models/StreamRecord`）
  - `Fields` は `Dictionary<string, JsonElement>` で保持。未知キーは破棄せずラウンドトリップで戻る
  - `JsonPropertyOrder` 属性で出力順を固定（要件のスキーマと一致）
- [x] JSON コンバータ／シリアライザ設定（`Persistence/Json/UlidJsonConverter` / `GameIdJsonConverter` / `JsonOptionsFactory`）
  - camelCase、`WriteIndented` （2 スペース）、UTF-8 BOM なし、日本語非エスケープ
  - `UnknownGameException` で対応外ゲーム検出
- [x] 状態機械（`States/AppStateMachine`）
  - 4 状態 + 3 サブ状態、不正遷移は `InvalidStateTransitionException`、`StateChanged` / `SubStateChanged` イベント
  - Recording を抜けるとサブ状態は自動クリア
  - 同一サブ状態への遷移はノーオペ（要件: エッジ検知時のみ副作用）
- [x] 永続化（`Persistence/JsonRecordStore`）
  - アトミック保存（tmp → fsync → 既存を .bak へ退避 → rename → .bak 削除、失敗時は .bak からロールバック）
  - `GenerateFileName(game, startedAt)` で `INFINITAS_yyyyMMdd_HHmmss.json` 形式
  - `Load` で `schemaVersion > Current` は `IncompatibleSchemaException`、破損は `InvalidDataException`
  - `FindUnfinished(directory)` で `stream.endedAt == null` をスキャン（破損ファイルは静かにスキップ）
- [x] Serilog 初期化（WPF 側、`App.xaml.cs`）
  - `logs/app_yyyyMMdd.log`、日次ローテ、7 日保持
  - `--log-level={trace|debug|info|warn|error|fatal}` 起動引数対応（エイリアス含む）
  - フォーマット: `[yyyy-MM-dd HH:mm:ss.fff] [LVL] [SourceContext] message`
- [x] テスト 40 件（モデル / JSON ラウンドトリップ / 永続化 / 状態機械 / ULID）

**DoD**:
- [x] `dotnet build`: 警告 0 / エラー 0
- [x] `dotnet test`: 40/40 合格
- [x] JSON ラウンドトリップ、アトミック保存後のファイル形状、状態遷移エッジ、復旧スキャナのテストが緑

**未実施**: アトミック保存の「書込途中で例外発生時にもファイルが破損しないこと」を実証するクラッシュシナリオテストは Phase 2 では作成せず（ファイル I/O モックが必要で重い）。`SaveAtomic` 内 catch でロールバックする実装にとどめた。Phase 6 直前に必要なら追加。

### Phase 3 — OBS 接続層と画面取得

**ゴール**: OBS と疎結合に接続でき、1Hz で `GetSourceScreenshot` を取得し続けられる。配信開始／終了イベントを購読し状態機械を駆動できる。

**作業**:
- `Obs/IObsConnection` 抽象（接続、Disconnect、配信状態イベント、`GetScreenshotAsync(string sourceName)`）
- `Obs/ObsWebSocketConnection` 実装（OBSWebSocketDotNet ラッパ）
- `Obs/ObsConnectionPool` 2 台 PC 構成の依存注入（StreamConnection と CaptureConnection を別インスタンスにできる）
- 接続切断時の自動再接続（1 → 5 → 10 → 30 → 60 秒バックオフ、状態ラベル更新）
- 1Hz 画面取得タスク（`PeriodicTimer` ベース）
- 取得失敗時は当該フレームをスキップ。連続 5 失敗で Warn ログ

**DoD**:
- OBS 実機接続が手元で確認できる（接続テスト UI は Phase 5 だが、ユニットテスト or 簡易コンソール harness で確認可能）
- モック `IObsConnection` でユニットテストが書ける（実 OBS なしで状態遷移テスト可能）

### Phase 4 — 画像認識

**ゴール**: ゲーム画面の連続フレームから状態判定とフィールド値抽出ができる。

**作業**:
- 解像度正規化（1920×1080 へのリサイズ／レターボックス／左右パディング）
- `Recognition/IHashMatcher`：aHash / pHash ベースの参照ハッシュ照合
- `Resources/INFINITAS/hashes.json` / `rois.json` の読み込み
- 状態判定（選曲中 / プレイ開始 / プレイリザルト、サブパターン束）
- 固定文言フィールド照合（diff_l / diff_s / lamp / dj_level）
- 数字 OCR（Tesseract、ホワイトリスト `0-9`）
- 楽曲タイトル OCR + 楽曲 DB fuzzy match（`Resources/INFINITAS/songs.json`、Levenshtein）
- エッジ検知時のタイムスタンプエントリ生成／更新ロジック
- デバッグ画像保存（Debug レベル時のみ、`logs/frames/{yyyyMMdd}/`、1000 枚超で古い順削除）

**着手前の前提**:
- `Resources/INFINITAS/` 配下の参照画像と JSON が必要。Python 版から流用 or 手動収集が必要。**画像認識方針の決定**メモリ（`memory/image_recognition_decision.md`）参照。
- 楽曲 DB は IIDX INFINITAS 楽曲一覧から作成。アプリ更新時の差し替えで追従。

**DoD**:
- ハッシュ照合・OCR・楽曲 fuzzy match の各ユニットテスト
- 1 配信分のキャプチャ画像列を入力にした統合テストが通る（参照データセットがあれば）
